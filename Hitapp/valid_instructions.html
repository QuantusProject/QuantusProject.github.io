We are trying to build a question-answering system which is able to retrieve one or more images to answer a user's query.

To explore this problem, we collected Questions, Answers to those Questions, and Images that imply those Answers. Now we want you to tell us a few things about each Question and Answer:

  1. Can you understand the Question without looking at an Image?
  2. Is this Answer really a valid and correct answer?
  3. Can this Answer be inferred from the Image(s)?
  4. Are at least two Images necessary to answer the Question?
  5. Do you think there is a mismatch between Image(s) and the Question-Answer pair, which might be caused by wrong index selection?
    i In our annotation task, we asked Turkers to select one or two image(s) from a group of images and then provide a question-answer pair based on the selected image(s). However, occasionally the image index(es) could be wrongly selected, which result in us losing track of the correct image(s). If you find that the question-answer pair is valid, but it seems so weird because the question-answer pair has nothing to with the image(s), then there might be a mismatch problem caused by wrong index selection. Select 'True' if you think there IS such a problem.

[NEW!]   Clarifications incorporating previous feedback:
  • No tolerance to spelling mistakes
    Capitalization error is fine. Underscores are allowed in names.
  • Questions must be interrogative
    De-qualify questions that are written as a declarative statement. Interrogative sentence with a missing question mark is fine.
  • If question is invalid, then there is no need to do the remaining validation for the same sample.
    In this case, the remaining selections will be automatically set to "False".
  • Answer lies in metadata ≠ inferred from image
    You can assume that the meta-data is there to help identify or locate the object of interest, as long as the identity or geographic information is not directly asked in the question. However, the property from which we directly draw the answer must be presented by some visual element in the image.
  • Select "False" when img does not load
    It rarely happens that some image sources have been deleted, so we need your help to filter out broken image links!
  • Be aware of fake multi-img questions
    For a valid multi-hop sample, the images should be complementary rather than substitutes for each other. In other words, both images should contain some information that the other does not contain. Questions that fail to satisfy this would fall into the single-hop bucket.
  • De-qualify questions that are easily answered by Google/about commonsense/too trivial
    Examples of such unqualified questions are like "Are there sidewalks in xxx city", "Are there buses on xxx street", etc.
  • De-qualify incoherent questions
    Questions that seem like two separate ones are not allowed.
  • How to handle binary questions with answers beyond a single "Yes/No"?
    The extra explanation beyond "Yes/No" will not by itself determine the validity of the answer. Essentially you can ignore the extra explanation and score the answer based on its "Yes/No"
